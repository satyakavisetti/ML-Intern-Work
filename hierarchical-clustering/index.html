Introduction

What hierarchical clustering is and how it builds clusters step by step.

What is it?

Hierarchical clustering creates a tree of clusters by either merging small groups or splitting a large one. It shows how data items relate across levels.

Why is it needed?

It is helpful when we want to see the structure of groups instead of one fixed solution. The tree can reveal natural clusters at different scales.

For students, hierarchical clustering is a strong way to understand how clusters grow or shrink and how the dataset is organized.

Real World Analogy

Think of organizing a library. Books start in broad categories, such as science or fiction, then move into subcategories. Hierarchical clustering shows the full structure from general to specific.

Problem it Solves

Choosing a fixed number of clusters can hide important relationships. Hierarchical clustering solves this by letting us inspect multiple possible groupings in one model.

  • Understanding nested cluster structure
  • Finding groupings at different levels of detail
  • Using a visual tree instead of a single flat partition

Mathematical Formula

Linkage options: distance between clusters determines how they merge.

d(A,B) = mina in A, b in B dist(a,b) (single linkage)

d(A,B) = distance between two clusters
dist(a,b) = distance between points
A, B = two clusters or groups
single linkage = uses the closest points between clusters

Algorithm Workflow

  1. Start with each point as its own cluster.
  2. Measure distances between every pair of clusters.
  3. Choose the closest pair and merge them.
  4. Update the distances using a linkage rule.
  5. Repeat until one cluster remains or until a stopping condition.
  6. Cut the tree at the desired level to form the final groups.

Working Example

Dataset

We have points from a set of handwritten digits. We want to see how the patterns cluster from general groups down to specific handwriting styles.

How it works

  1. Start with each sample as its own cluster.
  2. Merge the two closest samples first.
  3. Continue merging until the tree is complete.
  4. Cut the tree at a desired height to form clusters.

Advantages

Multiple resolutions

You can choose cluster detail after building the hierarchy.

Easy visualization

The tree structure reveals relationships clearly.

No k required

It does not require a fixed number of clusters at the start.

Limitations

Slow for large datasets

Computing all pairwise distances can be expensive.

Sensitive to noise

Outliers can affect the merge decisions and structure.

Choice of linkage matters

Different linkage rules produce very different trees.

Interview Questions

Divisive starts with one cluster and splits, while agglomerative starts with single points and merges.

Linkage is the rule used to compute distance between clusters when merging them.

A dendrogram shows the merge history and helps choose the cluster cut level.

Complete linkage tends to create tight and compact clusters.

It shows group structure at multiple scales instead of a single flat solution.

Summary

  • Hierarchical clustering reveals how groups form from individual points or larger clusters.
  • It is useful when we want a tree of possible clusterings rather than one fixed answer.
  • Distance and linkage rules shape the tree structure.
  • The dendrogram helps choose the right number of clusters.
